# Categories

Use categories to keep the marketplace browsable as it grows.

Recommended categories:

| Category | Use for |
| --- | --- |
| `development` | Coding, scaffolding, refactoring, architecture, and project setup |
| `workflow` | Git, pull requests, task management, release flows, and team process |
| `quality` | Code review, simplification, maintainability, and correctness checks |
| `security` | Defensive security review, secrets checks, threat modeling, and safe configuration |
| `testing` | Test generation, test debugging, coverage, and validation workflows |
| `documentation` | README, API docs, changelogs, writing, and knowledge-base workflows |
| `deployment` | Build, release, deploy, CI/CD, and operations workflows |
| `data` | Data analysis, transformation, reporting, and visualization workflows |
| `productivity` | General productivity tools and integrations |
| `experimental` | Early-stage plugins that are not ready for stable usage |

## Choosing a category

Pick the category that describes the plugin's primary user intent. If a plugin spans multiple areas, choose the category that best matches why a user would install it.

Avoid creating new categories until several plugins clearly need one.
